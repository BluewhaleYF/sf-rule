# sf-rule

这是一份专为 Shadowrocket 在 iPhone 上使用而精心调校的分流配置。针对移动端网络场景深度打磨，在保证海外服务可用性的同时，最大限度减少不必要的代理流量，延长电池续航，提升国内网页与应用加载速度。

配置文件同时附带 **轻量级广告屏蔽规则**，可拦截一些常见广告与追踪域名，让移动端浏览体验更清爽流畅。


## 安装

1. 打开 Shadowrocket 

2. 进入底部 **“配置”** 页面 

3. 点击右上角 **加号（+）**  

4. 在弹出菜单中选择 **“从 URL 下载”**，输入以下地址：

```
https://raw.githubusercontent.com/BluewhaleYF/sf-rule/refs/heads/main/sf-rule.conf
```

5. 点击 **“下载”**  

6. 下载完成后，点击 `sf-rule.conf`，选择 **“使用配置”**  

7. 返回 **“首页”**，将顶部 **“全局路由”** 设置为 **“配置”** 模式

## 规则特点

### 安全防护

- 防止 WebRTC 泄漏，保护隐私安全

- 使用国内 DNS 服务器加速访问，遇到问题时自动切换海外 DNS 避免 DNS 污染

- 本地地址不经过代理，确保本地服务使用正常

### 轻量广告过滤

屏蔽谷歌广告相关域名，使上网更顺畅，减少广告干扰并节省流量

请注意：仅能拦截部分已知的广告与追踪域名，**无法清除所有广告**。某些网站的内嵌广告、视频前贴片广告、原生广告等可能仍会正常展示。若您对广告屏蔽效果有更高要求，建议搭配专业广告拦截工具（如 AdGuard、1Blocker 等）共同使用。

### 国内直连优化

针对常见国内服务选择直连获取更低延迟和更佳速度

- 所有 `.cn` 域名

- **Bilibili**、**抖音**、**微信**、**QQ**、**网易**、**阿里巴巴** 等国内主流平台

- **Solsynth** 全系列产品域名

- **Apple 中国服务**（含 iCloud 及 CDN）

- **蔚蓝档案（国服）** 相关域名与 API

- **DeepSeek** 等国产 AI

### 海外完整代理

针对海外服务选择代理，特别优化细致的过滤策略，确保代理覆盖范围完整且规则匹配精准，能够很好地降低延迟

- **X**、**Facebook** 等社交软件

- **Google 服务**

- **YouTube**、**Spotify** 等流媒体平台

- **ChatGPT** 等国外 AI

- **Blue Archive 国际服（亚服）** 相关域名

- **GitHub**、**Notion**、**Telegram** 等海外平台

## 过滤原理

本配置的规则按照 **优先级从高到低** 依次排列，Shadowrocket 会从上到下逐条匹配，命中即停止。理解这一原理有助于您自行调整或排查问题。

### 匹配顺序

**请求目标域名 / IP**

  **↓**

**① 广告拦截（REJECT）**

匹配 `googleads`、`pubads` 等广告域名 → 直接拒绝连接

  **↓**

**② 国内直连（DIRECT）**

匹配 `.cn` 域名、Bilibili、抖音、Solsynth、Apple 中国服务等 → 直连，不经过代理

  **↓**

**③ 海外代理（PROXY）**

匹配 X、Google、YouTube、ChatGPT、GitHub 等海外服务 → 走代理

  **↓**

**④ IP 归属地检查（GEOIP）**

检查目标 IP 是否属于中国 → 是国内 IP 则直连，防止遗漏

  **↓**

**⑤ 最终兜底（FINAL）**

所有未匹配上述规则的请求 → 全部走代理

### 为什么这样设计？

| 优先级 | 策略 | 设计目的 |
|--------|------|----------|
| 1️⃣ REJECT | 广告域名最先拦截 | 节省流量、加快页面加载、保护隐私 |
| 2️⃣ DIRECT | 国内服务直连 | 避免代理绕路，降低延迟，节省代理流量 |
| 3️⃣ PROXY | 海外服务走代理 | 确保境外服务正常访问 |
| 4️⃣ GEOIP,CN | IP 层面补漏 | 捕获未在规则中显式声明的国内 IP，避免误走代理 |
| 5️⃣ FINAL | 全部代理 | 兜底处理未知域名，确保无遗漏 |

### 示例

以访问 `www.bilibili.com` 为例：

1. 检查是否匹配广告规则 → 否

2. 检查是否匹配直连规则 → ✅ 命中 `DOMAIN-SUFFIX,bilibili.com,DIRECT` → **直连**

以访问 `x.com` 为例：

1. 检查是否匹配广告规则 → 否

2. 检查是否匹配直连规则 → 否

3. 检查是否匹配代理规则 → ✅ 命中 `DOMAIN,x.com,PROXY` → **走代理**

以访问一个未在规则中列出的海外小站点为例：

1. 检查是否匹配广告规则 → 否

2. 检查是否匹配直连规则 → 否

3. 检查是否匹配代理规则 → 否

4. 检查 IP 是否属于中国 → 否

5. 最终兜底 → **走代理**

以访问一个未在规则中列出的国内站点为例：

1. 检查是否匹配广告规则 → 否

2. 检查是否匹配直连规则 → 否

3. 检查是否匹配代理规则 → 否

4. 检查 IP 是否属于中国 → ✅ 是国内 IP → **直连**

这样设计既保证了规则的可维护性，也最大程度减少了误判。
